### Target shape (webhook.ts onVoicemail, replacement block)
```typescript
// AMD reasons are all "picked up but is mailbox" variants; they all map to 'voicemail'.
// 'no_answer' / 'busy' fire from outbound-router error paths, not from AMD classifier.
// See RESEARCH §4.3.
type AmdVoicemailReason = 'amd_result' | 'cadence_cue' | 'silence_mailbox' | 'transcript_cue'
const amdReasonToPrevOutcome = (_r: AmdVoicemailReason): 'voicemail' => 'voicemail'

try {
  await coreMcpForAmd.callTool('voice_case_2_schedule_retry', {
    call_id: callId,
    target_phone: activeOutbound.target_phone,
    calendar_date: String(casePayload.requested_date ?? ''),
    prev_outcome: amdReasonToPrevOutcome(reason as AmdVoicemailReason),
    idempotency_key: String(casePayload.idempotency_key ?? ''),
  })
} catch (e: unknown) {
  log.warn({ event: 'case_2_schedule_retry_failed', call_id: callId, err: (e as Error)?.message })
}
```

### AMD reason → zod enum mapping table (RESEARCH §4.3, verbatim)

| `cadence_cue` | `voicemail` | Long uninterrupted speech = mailbox greeting |
| `silence_mailbox` | `voicemail` | Silent pickup = some IVR voicemail pattern |
| `transcript_cue` | `voicemail` | Regex matched mailbox phrase |
