# Phase 05.1 Plan 02: ASR Upgrade (whisper-1 → gpt-4o-mini-transcribe) Summary

One-liner: Upgraded the OpenAI Realtime session ASR model from whisper-1 to gpt-4o-mini-transcribe to fix structurally poor German transcription on 8kHz telephony audio (DEFECT #3), with a config-level regression test pinning the value.

## Context

Plan 05-03 Task 5 live-verification on 2026-04-20 surfaced DEFECT #3 (HIGH): whisper-1, even with `language='de'` pinned in commit 4db252c, still butchered short German phrases — "Hallo, hier Restaurant Bellavista" came back as "Jan-Uwe das war es von Bellevista" (DEFECTS §3). Two downstream failures followed: (1) the AMD classifier's transcript-cue regex CASE2_MAILBOX_CUE_REGEX_V2 cannot match on garbled text, and (2) the AMD model itself receives corrupted user input that may confound classification.

RESEARCH §3 documented gpt-4o-mini-transcribe as the 2026 OpenAI Realtime best-practice for German telephony audio — documented FLEURS WER improvement, drop-in compatible because voice-bridge consumes only `.completed` transcription events (§3.5 compatibility audit).

## What was built

**Task 1:** Swapped the transcription model literal in `voice-bridge/src/config.ts:306` from `'whisper-1'` to `'gpt-4o-mini-transcribe'`. Preserved `language: 'de'`. Expanded the adjacent comment block to cover (a) the Phase 05.1 defect-#3 rationale, (b) the drop-in compatibility basis (no `.delta` consumer), (c) fallback ladder (gpt-4o-transcribe → prompt-engineering → revert), and (d) Pitfall 3 cost-cap note referencing `CAP_PER_CALL_EUR=€1.00` in `src/cost/gate.ts`.

Created `voice-bridge/tests/config.test.ts` with two regression assertions:
- `SESSION_CONFIG.audio.input.transcription.model === 'gpt-4o-mini-transcribe'`
- `SESSION_CONFIG.audio.input.transcription.language === 'de'`
