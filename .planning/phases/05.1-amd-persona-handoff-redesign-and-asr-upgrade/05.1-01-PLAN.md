<objective>
Fix Defect #6 (CRITICAL): post-AMD persona handoff is silently broken because voice-bridge/src/sideband.ts:619 constructs session.update payloads without the required session.type='realtime' discriminator. OpenAI Realtime GA (2026) rejects such payloads with invalid_request_error, so CASE2_OUTBOUND_PERSONA never lands; the model keeps running on CASE2_AMD_CLASSIFIER_PROMPT and improvises a restaurant-assistant role from conversational context.

Layer 1 (primary): add type:'realtime' to session.update (sideband.ts:619, 1 character moved).
Layer 2 (defense-in-depth): inject synthetic conversation.item.create role=user directive in webhook.ts onHuman closure (between updateInstructions and requestResponse) to break conversational context inherited from AMD detector.
Observability: add explicit error-event handler in sideband.ts WS message loop (Pitfall 2 from RESEARCH.md §8).

Purpose: Unblocks Phase 05 Plan 05-03 Task 5 live verification (C2-03, C2-04, C2-05, C2-06 all depend on the bot actually being in CASE2_OUTBOUND_PERSONA role when talking to a human; C2-07 AMD gate is already working per DEFECTS.md §Observed AMD Classifier Successes).

Output: sideband.ts carries session.type='realtime' on every session.update; webhook.ts onHuman sends session.update→item.create→response.create in that order; sideband.ts logs session_update_rejected on any invalid_request_error. Wave 3 amd-classifier.ts and persona.ts content unchanged.
</objective>
