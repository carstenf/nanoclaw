OpenAI Python SDK contract (verified ≥1.51):
```python
from openai import OpenAI
client = OpenAI(webhook_secret=WEBHOOK_SECRET)
event = client.webhooks.unwrap(raw_body_bytes, request_headers)
# raises on bad signature; returns parsed event with event.type, event.id, event.data
```

FastAPI raw-body capture pattern (REQUIRED for HMAC):
```python
@app.post("/openai/webhook")
async def relay(request: Request) -> Response:
    raw = await request.body()  # bytes — DO NOT json.parse before unwrap
    event = client.webhooks.unwrap(raw, request.headers)
```

httpx forward signature:
```python
r = await request.app.state.http.post(
    BRIDGE_URL, content=raw,
    headers={k: v for k, v in request.headers.items()
             if k.lower().startswith("webhook-") or k.lower() == "content-type"}
)
```

Bridge endpoint contract (consumed by Plan 05):
- URL: http://10.0.0.2:4401/webhook
- Accepts: same raw body + webhook-* headers forwarded
- Returns: 200 on signature OK + log written; 401 on bad signature

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caddy loopback → forwarder | Trusted (same host) |
| Public internet → forwarder via Caddy | Untrusted; signature is auth |
| WG peer → forwarder canary | Authenticated by WG ChaCha20 |
| Forwarder → bridge over WG | Trusted by WG; bridge re-verifies sig anyway |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Forged webhook hits forwarder | mitigate | client.webhooks.unwrap() returns 401 on invalid HMAC (main.py); covered by test_invalid_signature_returns_401 |
| T-03-02 | Tampering | Body modified after signing | mitigate | HMAC over raw body bytes; await request.body() before any parsing |
| T-03-03 | Information Disclosure | Verbose error logs leak SIP From-headers | mitigate | log only event.type + size at INFO; Pitfall NEW-5 documented for future LOG_LEVEL=debug discipline |
| T-03-04 | DoS | Public flood at /openai/webhook | mitigate | Caddy + cloud firewall absorb; OpenAI rate is bounded; Pitfall NEW-1 firewall blocks direct 9876 |
| T-03-05 | Spoofing | Direct POST to 127.0.0.1:9876 bypasses Caddy | accept | Same code path; signature still required; Caddy is just TLS terminator + path filter |
| T-03-06 | Repudiation | No request log for accepted webhooks | mitigate | INFO log entry with event_type, size on every accept |
| T-03-07 | Elevation of Privilege | Container runs as root | accept | python:3.12-slim default; Phase 4 hardening can add USER directive; not a Phase 1 blocker |
</threat_model>
