<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OpenAI → public internet → Caddy | Untrusted webhook deliveries; signature is the auth |
| Caddy → forwarder loopback | Trusted (same host) |
| Secret-at-rest in .env files | Trusted (filesystem perms) |
| Sipgate carrier media gateway → Hetzner public UDP 60000-60100 | RTP media; carrier-trusted (no SRTP in PCMU classic SIP); firewall must allow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Forged webhook to Caddy public URL | mitigate | Forwarder + bridge re-verify HMAC via openai.webhooks.unwrap() (Plans 03 + 05); attacker without secret cannot forge |
| T-02-02 | Information Disclosure | OPENAI_WEBHOOK_SECRET leak from .env | mitigate | chmod 600 on both .env files; transfer via OneCLI vault or password-manager (not chat); .env in .gitignore |
| T-02-03 | Tampering | Webhook URL hijack via DNS | accept | DNS owned by carsten; out-of-scope for this phase; ACME validates ownership at TLS issuance |
| T-02-04 | Information Disclosure | Caddy snippet exposes /__wg_canary or /health publicly | mitigate | `handle @webhook path /openai/webhook` + catch-all `handle { respond 404 }` per Template 7 (Assumption A5) |
| T-02-05 | DoS | Public POST flood at /openai/webhook | accept | OpenAI Standard Webhooks rate is bounded; Caddy can absorb; revisit if observed |
| T-02-06 | DoS | Open RTP port range 60000-60100 attracts UDP scan / amplification | accept | FreeSWITCH only accepts RTP within established SIP dialogs; orphan UDP packets dropped at app layer; standard practice for SIP carriers; tightening to Sipgate carrier IPs deferred to Phase 4 hardening |
</threat_model>
