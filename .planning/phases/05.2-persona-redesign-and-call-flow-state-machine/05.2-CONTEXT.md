# Phase 05.2 CONTEXT — Persona Redesign + Call-Flow State-Machine

## Scope anchor

Phase 05.2 implements the architecture recommended in `.planning/research/voice-persona-architecture.md` §6.1 ("Baseline + Task overlay with state transitions"), mapped to the Phase 4 tier (baseline + per-case overlay via `session.update`). Phase 5 (full state-machine / Pipecat-Flows-style) is out of scope — reserve for later if case count grows beyond 5-6.

Three concrete issues from Phase 05.1 live-verification drive this phase:
1. Role-hallucination on Case-1 outbound persona (missing role-lock clause)
2. Premature silence re-prompt (state-machine bug: timer not bot-audio-aware)
3. Outbound bot-speaks-first (create_response:true at session-accept time)

Researcher and Planner: read `voice-persona-architecture.md` §1-§6 first — it contains the concrete German baseline persona draft (~515 tokens, §6.2) and Case-2 overlay example (~200 tokens, §6.3). Treat this CONTEXT.md as overlay for locked decisions that narrow the research's open questions.

## Decisions (locked)

### D-1. Re-prompt attempt count = 3

Source: OpenAI Realtime Cookbook "3 consecutive no-match/no-input events" escalation pattern (voice-persona-architecture.md §3.3, §6.4 Q1). Tunable per-case via env override for future per-case tuning, but initial baseline ships with 3 uniform across all cases. Carsten's original spec of 2 was aggressive; 3 matches industry default (LiveKit, Pipecat, OpenAI Cookbook) and gives Carsten himself more tolerance when inbound calls are received while he's distracted.

### D-2. Farewell phrasing — apologetic Sie-form outbound, Du-form inbound Carsten

Outbound final-nudge farewell (after 3 failed attempts):
> "Ich erreiche Sie gerade nicht, ich versuche es später nochmal. Auf Wiederhören."

Inbound Carsten final-nudge farewell:
> "Ich melde mich später nochmal, Carsten — tschau!"

Cookbook Role&Objective pattern: maintain respectful tone to unknown counterparts (Sie-form), casual with known owner (Du-form). Caller-number whitelist selects which. Replaces current `OUTBOUND_PERSONA_TEMPLATE`'s neutral-accusatory *"Ich lege jetzt auf, es ist niemand mehr da."* (voice-persona-architecture.md §3.4, §6.4 Q2).

### D-3. Architecture tier = Phase 4 (Baseline + Task Overlay via session.update)

NOT Phase 5 full state-machine (Pipecat-Flows style). Rationale: Phase 4 gets ~66% token reduction at ~30% implementation complexity per research §6.1. Phase 5 adds runtime complexity and multi-state transitions not justified at current case count (5 cases planned: Case 2 restaurant, Case 3 medical, Case 6b inbound Carsten, Case 1 general outbound, AMD-classifier-mode). If case count grows beyond ~8, revisit Phase 5.

Single `session.update` per call for the baseline→task overlay handoff. The AMD-handoff pattern already implemented in `voice-bridge/src/webhook.ts:431-475` (Phase 05.1) is the reference implementation — reuse its mechanics.

### D-4. Anrede-form source = caller-number whitelist (unchanged)

Current behavior in persona-selector logic picks Sie vs Du based on whether caller_number is in the Carsten-whitelist. Keep this mapping. Baseline persona accepts `{{anrede_form}}` and `{{counterpart_label}}` as template variables; the lookup stays server-side, not per-call-field. No new bridge signals needed (voice-persona-architecture.md §6.4 Q4).

### D-5. Cost-cap verification = parallel, non-blocking

gpt-realtime was deployed 2026-04-21 but CAP_PER_CALL_EUR=€1.00 / CAP_MONTHLY_EUR=€25.00 have not been stress-tested at the new ~3x per-minute rate. Plan 05.2 ships without a blocking cost-verify call. The first Phase 05.2 live verification call exercises both the new persona AND the cost caps; any cost regression surfaces in the verification report and triggers a separate fix plan if needed (voice-persona-architecture.md §6.4 Q5).

### D-6. Inbound self-greet signal = fixed 1000ms setTimeout after /accept (unchanged)

Current behavior satisfies user's "1-2 sec selbst melden" spec per `voice-persona-architecture.md §6.4 Q6`. Option B (wait for first caller-RTP packet) is orthogonal and can be added later if the 1000ms fixed delay creates awkward overlap with counterpart's own greeting. No new bridge signals needed in this phase.

### D-7. silence-monitor bug fix = state-machine rewire, NOT prompt change

Fix silence-monitor.ts: arm silence countdown on `output_audio_buffer.stopped` (bot finished speaking) — NOT caller `speech_stopped`. Cancel on `output_audio_buffer.started` (bot speaking) OR caller `speech_started`. Re-arm on whichever bot/caller event comes next. This is a state-machine change, not a persona-prompt change. Events are OpenAI Realtime server-events; reference `voice-bridge/src/sideband.ts` existing handlers for integration pattern (voice-persona-architecture.md §4.2).

### D-8. Outbound wait-for-speech = create_response:false + manual response.create

Flip `SESSION_CONFIG.audio.input.turn_detection.create_response` from `true` to `false`. Bot must explicitly receive a `response.create` sideband event before first utterance. Two paths:
- Outbound: bot stays silent until caller's first `speech_stopped` fires (counterpart said something); then fires response.create.
- Outbound nudge ladder: if no caller speech within 5s post-accept, bot fires response.create with "Hallo, ist da jemand?" nudge.
- Inbound: bot fires response.create at self-greet-timeout (1000ms post-/accept, per D-6).

Research §3.1-§3.2 documents this pattern; native OpenAI Realtime support confirmed. This closes Carsten's "bot sollte warten bis sich jemand meldet" requirement (voice-persona-architecture.md §6.4 Q3 — not explicitly an open question but the authoritative research conclusion).

### D-9. Role-lock clause in baseline persona (NEW text)

Add explicit role-discipline clause to the baseline persona (ref `voice-persona-architecture.md §5.2` role-lock excerpts from 3 vendors). Verbatim text candidate:

```
TURN-DISCIPLIN:
  - Du sprichst NUR deine eigene Rolle. Du erfindest NIEMALS, was die Gegenseite sagt.
  - Warte auf eine echte Antwort der Gegenseite, bevor du weiterredest.
  - Wenn du unsicher bist, ob die Gegenseite gesprochen hat: schweige und höre.
```

This block is NEW — neither the current OUTBOUND_PERSONA_TEMPLATE nor CASE6B_PERSONA contains equivalent language. Required specifically to stop the role-hallucination observed on gpt-realtime-mini (persists at lower rate on gpt-realtime). Addition applies to all cases because it's baseline-level.

### D-10. Phase 05.1-05 live verification = deferred to AFTER 05.2 lands

Plan 05.1-05 (the 3-PSTN-call live verification from Phase 05.1) is explicitly deferred. Running it with the known-broken Case-1 persona would burn telephony minutes on predictably-bad traces. The 05.2 architecture fixes all three issues (role-play, silence-timing, outbound bot-first) — post-05.2, we run ONE combined live-verification that covers both the 05.1 defect fixes AND the 05.2 redesign.

Open item tracked in ROADMAP.md Phase 05.1 entry; this phase's final plan references it.
