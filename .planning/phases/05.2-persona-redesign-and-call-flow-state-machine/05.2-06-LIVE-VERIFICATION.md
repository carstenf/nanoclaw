## Root cause (unchanged from Phase 05.1 DEFECTS.md findings)

The CASE2_AMD_CLASSIFIER_PROMPT instructs the model to emit `amd_result{verdict:human}` when a human greets. The model (gpt-realtime, upgraded from mini in Plan 05.1-02) does NOT reliably emit this function_call even for prompt-matching inputs. In Phase 05.1 live testing (2026-04-20) the same issue surfaced — only 1 of 3 calls produced `verdict=human` (DEFECTS.md §Observed AMD Classifier Successes).

Compounding issue: `voice-bridge/src/amd-classifier.ts` has an unused `onSpeechStarted()` handler that would cancel Timer B when a caller's VAD fires `speech_started`. This wiring does not exist in sideband.ts — `speech_started` events only route to `silence-monitor.onSpeechStart` (call-router.ts:125). The classifier is effectively blind to caller VAD events, relying entirely on the model's function_call behavior (which is unreliable).
