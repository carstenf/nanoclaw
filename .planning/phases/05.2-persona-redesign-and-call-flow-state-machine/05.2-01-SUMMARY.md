## Decisions Made

- **Verbatim-copy-over-paraphrase:** BASELINE_PERSONA_TEMPLATE is a literal transcription of research §6.2 lines 567-644. No rewording, no re-structuring — zero-drift reproducibility. Anyone tracing the test marker "Rolle (KRITISCH)" back from runtime to research doc finds identical text.
- **ASCII-umlaut convention enforced:** "Gegenueber", "erfinde", "unterwuerfig", "Geraeusche" (Phase 2 CASE6B_PERSONA truths[8] project-wide rule). Preserved against UTF-8 umlauts to avoid terminal/log pipeline corruption observed in earlier phases.
- **Overlay bodies deferred to 05.2-04:** Wave 1 is purely additive (new files only). Legacy `voice-bridge/src/persona.ts` remains byte-identical; migration of buildCase2OutboundPersona into the case_2 overlay lands in Plan 05.2-04. This allows 05.2-02 (silence-monitor rewire) to run in parallel with zero file conflict.
- **Anrede derivation table computed in factory:** Caller surface is a single `anrede_form: 'Du' | 'Sie'` enum. Factory derives anrede_capitalized (dich/Sie), anrede_pronoun (du/Sie), anrede_disclosure (Bist du/Sind Sie) for the re-ask phrase + disclosure phrase. Research §6.2 used `{{Sie|dich}}` pipe-pair shorthand which is not a real template-engine construct — explicit substitution is auditable.
- **CaseKey typed at compile time:** Typo-safe registration. Wave 2/3 call sites (outbound-router, webhook.ts) will get compile-time errors if they drift from the canonical set of case keys.
