<objective>
Research-deliverable plan (NO production-code changes). Answer the four D-4 questions
about OpenAI Realtime `turn_detection.idle_timeout_ms` behavior before Plan 05.3-05
writes the timer-removal code. Produces a single authoritative finding file at
`.planning/phases/05.3-refactor-cleanup-timer-removal/idle-timeout-finding.md` that
Plans 05.3-05a and 05.3-05b read as their first context input.

Purpose: The CONTEXT D-3 setTimeout removal strategy hinges on whether OpenAI's
native `idle_timeout_ms` actually fires `response.create` server-side with
conversation history + instructions in scope, and whether it coexists with the
`create_response:false` wait-for-speech contract set by Plan 05.2-03 D-8. Getting
this wrong turns Plan 05.3-05a/05b into a regression (silent bot or early-audio §201
violation). A short research pass resolves this before code moves.

against a live OpenAI Realtime session if the docs are ambiguous — Carsten runs the
probe if needed (same pattern as the Plan 05.2-05 Q7 atomicity probe). If docs alone
suffice, the plan upgrades to fully autonomous on its own — executor decides after
initial docs scan.
