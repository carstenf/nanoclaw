## Deviations from Plan

**Plan executed as written. Two minor observations documented for transparency:**

1. **Plan Task 1 verify regex quirk (Rule 1 — documented, not blocking):** The plan's `<verify>` automated check was `grep -qE 'Verdict:\s*(FULLY-DELETE|SHRINK-TO-HARD-SAFETY-STUB|RETAIN-AS-IS)' idle-timeout-finding.md`. The actual finding file line 221 reads `**Verdict:** SHRINK-TO-HARD-SAFETY-STUB` — `\s*` matches the space AFTER `**` but the grep never gets there because it requires `:` then whitespace (the `**` between `:` and space breaks the match). Using `grep -qE 'Verdict:[* ]+SHRINK-TO-HARD-SAFETY-STUB'` confirmed the verdict unambiguously. The PLAN's regex needs Markdown-bold-aware `:.{0,5}` for future checks; the verdict itself is crystal-clear. No functional impact.

2. **sideband.ts opts.onBotStart?.() / opts.onBotStop?.() left in place (Rule 1 — scoped):** PLAN Task 3 STEP 2 mentioned the option to "delete the handlers AND their `opts.onBotStart/onBotStop` consumer in sideband.ts event dispatch" if they become dead. In this worktree:
   - `call-router.ts` no longer passes `onBotStart`/`onBotStop` to sideband.
   - `sideband.ts` still emits `opts.onBotStart?.()` / `opts.onBotStop?.()` optionally — harmless no-op when caller omits the callback.
   - The event dispatch itself (on `output_audio_buffer.started` / `stopped`) is trivial (2 lines per event); removing it would touch sideband.ts + its types.
   - Left as-is to keep the blast radius minimal (plan `files_modified` frontmatter lists call-router.ts + silence-monitor.ts + 2 test files — sideband.ts not in scope). Out-of-scope cleanup logged for Plan 05.3-06 final-sweep consideration.
