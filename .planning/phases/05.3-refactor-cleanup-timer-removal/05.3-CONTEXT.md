### D-3. Timer removal via OpenAI Realtime native `idle_timeout_ms`

Replace all server-side UX timers with OpenAI Realtime's `turn_detection.idle_timeout_ms` native feature (Research §4 referenced this). Scope:

**Remove:**
- `GREET_TRIGGER_DELAY_OUTBOUND_MS` (2500ms setTimeout at webhook.ts:473-475 Case-2 onHuman). Replace with: synchronous `requestResponse` after session.update+conversation.item.create (per Plan 05.2-05 Q7 finding: session.update is docs-lean ATOMIC).
- `GREET_TRIGGER_DELAY_MS` (1000ms setTimeout for inbound self-greet at webhook.ts:725). Replace with: inbound SESSION_CONFIG sets `turn_detection.idle_timeout_ms=1000` for self-greet, OR fire `response.create` synchronously post-accept (bot greets immediately — caller-number whitelist means inbound is always Carsten).
- `silence-monitor.ts` 10s round timer + 3-round re-prompt ladder (lines ~68-135 of silence-monitor.ts, 166 logic-lines total). Replace with: `turn_detection.idle_timeout_ms` triggers native `response.create` from OpenAI. Persona's OUTBOUND_SCHWEIGEN/INBOUND_SCHWEIGEN blocks (already in baseline from Plan 05.2-03) guide the model's nudge text.

**Keep (HARD-safety floor):**
- Hangup-after-N-minutes cost-cap protection — evaluate whether silence-monitor needs a small residual layer OR if cost/gate.ts already covers this independently. Prefer: delete silence-monitor entirely if cost-gate covers.
