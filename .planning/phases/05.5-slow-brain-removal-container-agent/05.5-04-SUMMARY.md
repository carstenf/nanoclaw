- **call-router.ts onTranscriptTurn branch (REQ-DIR-06):** transcript events branch on REASONING_MODE. Container-agent fires `nanoclawMcp.transcript({...})` fire-and-forget per D-12; result.instructions_update flows through existing `updateInstructions(state, ...)` session.update pusher. Slow-brain default uses legacy `slowBrain.push({turnId, transcript})`. AMD VAD-fallback (`getAmdClassifier()?.onTranscript`) preserved in BOTH modes (Plan 05.2-06 invariant).
- **CallContext shape extended:** new optional `nanoclawMcp` field + new always-present `turnHistory: TurnHistoryEntry[]` (initialized to `[]`). counterpart turns appended in onTranscriptTurn before the branch (so MCP call sees up-to-date history). `coreMcp` and `slowBrain` fields preserved during rollout — Plan 05.6 cleanup deletes them.
- **webhook.ts Splice A (outbound /accept):** non-Case-2 outbound paths branch on REASONING_MODE. Container-agent calls `nanoclawMcp.init({call_id, case_type:'case_2', call_direction:'outbound', counterpart_label:<restaurant_name|'Counterpart'>})` synchronously per REQ-DIR-19, uses returned `instructions`. On timeout/error → FALLBACK_PERSONA (REQ-VOICE-13 fallback). Case-2 keeps `CASE2_AMD_CLASSIFIER_PROMPT` at /accept (§201 AMD-gate invariant); persona swap happens via existing onHuman path post-verdict.

- **Config-error guard `container_agent_mode_but_client_missing`:** if REASONING_MODE='container-agent' but `nanoclawMcp` undefined (e.g. NANOCLAW_VOICE_MCP_URL not set), log error + use FALLBACK_PERSONA. Never silently regress to legacy code.

Slow-brain default leaves it `undefined` → no MCP connection attempted. Plumbed through `registerAcceptRoute(...)` 8th arg + `BuildAppOptions.nanoclawMcpOverride` for tests.
- **Per-call lifecycle:** `nanoclawMcp` plumbed through both `router.startCall` call-sites (outbound + inbound) so call-router stashes it on CallContext. endCall finalizer calls `ctx.nanoclawMcp?.close()` idempotently alongside existing `ctx.slowBrain.stop()`.

- **Issue:** Plan-pattern verbatim placed Case-2 outbound under `nanoclawMcp.init` at /accept. But Case-2 needs `CASE2_AMD_CLASSIFIER_PROMPT` (the listen-only AMD classifier prompt) as initial /accept instructions — not a normal persona — so the model stays silent until the AMD verdict (§201 StGB / Plan 05-03 T-05-03-01). - **Fix:** Narrowed Splice A to `useContainerAgent = REASONING_MODE === 'container-agent' && !isCase2`. Case-2 keeps its existing classifier-prompt path at /accept; the persona swap that already lives in the `onHuman` closure (post-AMD-verdict) remains the legacy Phase-5 path. Plan 05.6 will swap the onHuman path to also call the container-agent at that point.

- **Verification:** existing `webhook-amd-handoff.test.ts` (5 tests) + `webhook-case-2-voicemail.test.ts` (5 tests) all still green; new branch test covers non-Case-2 outbound under container-agent.
