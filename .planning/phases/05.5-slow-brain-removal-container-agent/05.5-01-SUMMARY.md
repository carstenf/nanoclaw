# Phase 05.5 Plan 01: VoiceTriggerQueue + voice_triggers_init/transcript MCP-tools + gc Hook — Summary

NanoClaw-Side foundation für die Container-Agent-Reasoning-Layer: per-`call_id` FIFO Promise-chain queue, zwei neue MCP-tools auf Port 3201 mit no-op DI-Stubs (D-24 Scope-Boundary), end-of-call gc-Hook in `voice_finalize_call_cost`.

## VoiceTriggerQueue API surface

`src/voice-trigger-queue.ts` — slim, in-process, per-key Promise-chain queue.

```ts
class VoiceTriggerQueue {
  enqueue<T>(callId: string, fn: () => Promise<T>): Promise<T>;
  gc(callId: string): void;
  depth(callId: string): number;
}
```

Invarianten (alle in `voice-trigger-queue.test.ts` per Test abgedeckt):

| Invariant | Mechanismus |
|-----------|-------------|
| FIFO innerhalb eines `call_id` | `prev.then(fn, fn)` chain |
| Concurrency über verschiedene `call_id` | unabhängige Promise-Chains pro Key |
| Failing task vergiftet Chain nicht | `prev.then(fn, fn)` (D-12) |
| `gc()` löscht Chain + Depth (REQ-INFRA-16) | `Map.delete` |
| `depth()` reflektiert in-flight + pending | counter inkrementiert in `enqueue`, dekrementiert in `next.finally` |

Keine externen Dependencies. Kein container-process state. Kein retry.

## gc-on-Finalize-Hook (REQ-INFRA-16, D-11)

`src/mcp-tools/voice-finalize-call-cost.ts` ruft am Ende des Success-Paths:

```ts
const indexMod = (await import('./index.js')) as {
  voiceTriggerQueue: { gc: (callId: string) => void };
};
indexMod.voiceTriggerQueue.gc(d.call_id);
```

**Dynamic-import-Pattern**: `mcp-tools/index.ts` importiert `voice-finalize-call-cost.ts` schon statisch (für `makeVoiceFinalizeCallCost`), eine statische Reverse-Import-Linie würde loop'en. Dynamic Import löst die zyklische Abhängigkeit zur Laufzeit auf und failed graceful (loggt `voice_trigger_queue_gc_failed`, der nächste `gc()` wäre idempotent).

Test in `voice-finalize-call-cost.test.ts`: seedet einen in-flight Task in den Singleton, ruft den Handler, asserted `depth(call_id) === 0` nach Finalize. End-to-end Verification durch echten Singleton — kein Mock.
