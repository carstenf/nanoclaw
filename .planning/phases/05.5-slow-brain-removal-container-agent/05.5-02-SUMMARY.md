## SCHWEIGEN-Ladder Convention Chosen

Inline conditional blocks inside `baseline.md`, delimited by HTML comments:

```
<!-- BEGIN SCHWEIGEN_LADDER call_direction=inbound -->
... INBOUND_SCHWEIGEN body ...
<!-- END SCHWEIGEN_LADDER -->
<!-- BEGIN SCHWEIGEN_LADDER call_direction=outbound -->
... OUTBOUND_SCHWEIGEN body ...
<!-- END SCHWEIGEN_LADDER -->
```

The container-agent substitutes the matching block for the `{{SCHWEIGEN_LADDER}}` token at render time (algorithm documented in SKILL.md `## Placeholders > {{SCHWEIGEN_LADDER}} direction-conditional convention`). The non-matching block is dropped from the rendered persona — same isolation guarantee as the legacy `args.call_direction === 'outbound' ? OUTBOUND_SCHWEIGEN : INBOUND_SCHWEIGEN` ternary in `baseline.ts:170`.

Rejected alternative: external `overlays/_schweigen-{inbound,outbound}.md` partials. The inline approach keeps all baseline content in one file (easier spot-check), and the existing `{{SCHWEIGEN_LADDER}}` placeholder name maps cleanly to a single substitution slot.
