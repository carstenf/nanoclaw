| T-4.5-E | Information Disclosure / Resource leak (session leak on call end) | sideband.ts CallSession finalizer | **mitigate** | Per-call CoreMcpClient held in CallSession; on `session.closed` / BYE / hangup path, finalizer awaits `coreClient.close()` inside try/catch (so a close failure logs but doesn't prevent other teardown steps). log event `bridge_core_mcp_client_closed` on success, `bridge_core_mcp_client_close_failed` on error. Regression test `reconnect_after_close` from 04.5-02 proves idempotency. |

Bridge per-call pattern generates O(N_calls) sessions; at peak voice load (~1 call at a time per Case 6), practical session count ≪ 50. If load changes, adjust `MCP_STREAM_MAX_SESSIONS`. |
