<!-- OneCLI-Proxy fuer Host-Prozess (nicht Container) -->
OneCLI ist ein MITM-HTTPS-Proxy. Fuer Container erledigt `applyContainerConfig` Proxy-Env + CA-Bundle-Mount. Fuer einen HOST-Prozess (NanoClaw-Core laeuft nativ) muss man den Proxy manuell dispatchen. Node 18+ undici bietet `ProxyAgent`.

```typescript
import { ProxyAgent } from 'undici'
const dispatcher = new ProxyAgent(ONECLI_URL)
const res = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'anthropic-version': '2023-06-01',
  },
  body: JSON.stringify({
    model: 'claude-sonnet-4-5-20241022',
    max_tokens: 300,
    system: SYSTEM_PROMPT,
    messages: [...],
  }),
  dispatcher, // undici ProxyAgent routes via OneCLI
  signal: abortCtrl.signal,
})
```

OneCLI intercepts the TLS handshake, provides its own CA cert (need to trust it — existing NanoClaw already has OneCLI CA in system CA bundle per container-runner.ts pattern, same host so trust exists).

<!-- Anthropic Messages API shape -->
POST /v1/messages
body: {
  model, max_tokens, system, messages: [{role, content}]
}
response: {
  content: [{type:'text', text}], ...
}

Kein API-Key-Header noetig — OneCLI injects on MITM.

<!-- NanoClaw-CA-Bundle bereits vorhanden (container-runner.ts Zeile ~280-320) -->
Gleicher Host = gleicher CA-Trust fuer den Core-Prozess. Keine extra-Config noetig; undici picks up `NODE_EXTRA_CA_CERTS` wenn gesetzt. Falls im Test-setup nicht gesetzt: systemd-unit erweitern.
