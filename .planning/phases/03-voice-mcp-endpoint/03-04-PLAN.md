  const jid = `dc:${channelId}`
  const channel = findChannel(channels, jid)
  if (!channel || channel.name !== 'discord') {
    logger.warn({channelId}, 'Discord channel not found or not discord-type')
    return  // callback resolves; handler will detect no-throw and return delivered=true? NO — see truths[2] — we need to propagate "discord_not_configured"

}

Wichtig: wenn Discord nicht connected ist (kein BotToken gesetzt), `findChannel` returned `undefined`. In dem Fall muss der Handler `{ok:false, error:'discord_not_configured'}` returnen — deshalb **throwt** der callback in dem Fall ein sentinel-Error (`Error('discord_not_configured')`), der Handler catched das.

Revision: statt throw → callback returned boolean? Simpler: callback signature `(channelId, text) => Promise<{ok: true} | {ok: false, error: string}>`. Dann kann der Handler das direkt mappen.

**Finale Signatur:**
```typescript
sendDiscordMessage: (channelId: string, text: string) => Promise<{ok: true} | {ok: false, error: string}>
```
