<objective>
Close the four empirical unknowns that gate Wave 3 design decisions. OQ-1 (can the
Realtime model emit a function_call as first output?), OQ-3 (exact Sipgate 486 body
shape), A3/A5 (German mailbox greeting corpus for cue-regex hardening), and OQ-5
(travel-buffer source address) are all cheap to resolve empirically but expensive
to guess wrong. Wave 0 is human-in-the-loop: Carsten places live PSTN calls from
his second phone under controlled conditions; carsten_bot collects logs, parses
outputs, and writes four SPIKE-*.md evidence files. Output is data + documented
decisions, not production code. The only production code touched is a small
sipgate-rest-client.ts patch to log the full error body verbatim so Spike-B can
capture it without instrumentation debt.

Purpose: Prevent Wave 3 from rebuilding AMD from scratch if OQ-1 is rejected, and
prevent Wave 2 from guessing at Sipgate error parsing. Removing these unknowns is
worth one wave's effort because the cost of being wrong mid-Wave-3 is a full
re-plan.

Output: Four evidence files under spike-results/, a mailbox-greeting fixture
directory (text transcripts only — no audio persistence per §201 StGB), and one
minor sipgate-rest-client.ts instrumentation patch that is kept in production.
</objective>

Spike-A classifier prompt (verbatim from 05-RESEARCH.md §2.4):

```
Du bist in einem Detektions-Modus. Der Anruf wurde GERADE angenommen.
Deine EINZIGE Aufgabe ist: bestimme, ob ein Mensch oder eine Mailbox/Anrufbeantworter angenommen hat.

KRITISCH: Du sprichst JETZT NICHT. Generiere KEIN Audio. Du hörst nur zu.

Höre die ersten 3 Sekunden:
- Wenn ein Mensch knapp grüßt → emit function_call "amd_result" with arg {"verdict": "human"}
- Wenn eine Ansage läuft → emit function_call "amd_result" with arg {"verdict": "voicemail"}
- Wenn 4 Sekunden lang NICHTS gesprochen wird → emit function_call "amd_result" with arg {"verdict": "silence"}

Sprich NIEMALS bis die Bridge dir neue Anweisungen gibt.
```

Spike-A tool shape (temporary, Bridge-scratch only — NOT added to allowlist.ts permanently):

```json
{
  "name": "amd_result",
  "parameters": {
    "type": "object",
    "properties": { "verdict": { "enum": ["human", "voicemail", "silence"] } },
    "required": ["verdict"]
  }
}
```

Spike-A success predicate:
  - response.audio.delta count before amd_result function_call = 0
  - amd_result fires within 4000ms of realtime.call.incoming (after pickup)
  - at least 3 trials per scenario (human/voicemail/silence)

Spike-B success predicate:
  - Captured HTTP status + JSON body for:
      Scenario 1: target line busy (second phone off-hook)
      Scenario 2: target line 408 timeout (ring forever, Sipgate gives up)
  - Response body JSON keys enumerated; if a `causeCode` / `status` / similar field exists, it is documented

Spike-C success predicate:
  - fixtures/mailbox-greetings/ contains ≥10 .txt files with distinct German mailbox greetings
  - SPIKE-C-mailbox-corpus.md has a coverage table: greeting # | matches CASE2_MAILBOX_CUE_REGEX (yes/no) | proposed regex patch if no

Spike-D success predicate:
  - SPIKE-D-travel-buffer-source.md chooses ONE of: (a) static config CASE_2_TRAVEL_SOURCE_ADDRESS in src/config.ts, (b) per-call arg source_address in voice_start_case_2_call schema, (c) defer travel buffer entirely to follow-up phase (no travel buffer in Case-2 MVP). Decision is supported by CONOPS reference + MOE-6 risk check.
